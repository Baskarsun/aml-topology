 ---
  AML TOPOLOGY DETECTION SYSTEM — COMPLETE PIPELINE ANALYSIS

  1. SYSTEM OVERVIEW

  Platform: FRAUD.GUARD | AML Detection Platform
  Stack: Streamlit (UI) · Flask (API) · PyTorch (LSTM/GNN/Sequence) · LightGBM/XGBoost (GBDT) · NetworkX (Graph) · SQLite (Persistence)

  The system implements 8 detection phases that feed into a weighted risk consolidator, producing per-account risk scores surfaced through a Streamlit dashboard and REST API.

  ---
  Phase 4 — LSTM Link Prediction (src/lstm_link_predictor.py)

  Architecture:
  (N, seq_len=3, features)  →  LSTM(hidden=64, layers=1, dropout=0.1)
                             →  FC(hidden → 1)
                             →  Sigmoid → binary link probability

  Node Embeddings (src/embedding_builder.py):
  - Static (7 dims): in/out/total-degree, PageRank, clustering coefficient, reciprocal count, unique counterparties
  - Dynamic per 12h bucket (13 dims): tx count in/out, total/avg/median/std amount in/out, unique targets, time-since-last-tx
  - Fraud score (1 dim): Historical flag

  Training: BCEWithLogitsLoss + Adam (lr=1e-3), pos_weight for class imbalance, 15 epochs, 10% val split

  Output: Top-10 emerging suspicious links + probability scores

  ---
  Phase 5 — GBDT Transaction Scorer (src/gbdt_detector.py)

  Backend: LightGBM (primary), XGBoost / CatBoost (fallback)

  Feature Set (12 features):
  - amount, mcc, payment_type, country
  - device_change (bool), ip_risk (float 0–1)
  - count_1h (velocity), sum_24h, unique_payees_24h
  - Derived risk rules: amount > 10000 AND device_change, payment_type='crypto' AND amount > 5000

  Training: 20,000 synthetic samples, fraud_rate=3%
  Persistence: models/lgb_model.txt

  ---
  Phase 6 — Sequence Detector (src/sequence_detector.py)

  Two model variants:

  ┌─────────────┬──────────────────────────────────────────────────────────────────────────┬──────────────────────┐
  │    Model    │                               Architecture                               │      Key Params      │
  ├─────────────┼──────────────────────────────────────────────────────────────────────────┼──────────────────────┤
  │ LSTM        │ Event embed(32) + scalar projection → LSTM(64) → 2-layer FC              │ dropout, batch_first │
  ├─────────────┼──────────────────────────────────────────────────────────────────────────┼──────────────────────┤
  │ Transformer │ Event embed(32) → MultiheadAttn(nhead=4, nlayers=2) → adaptive pool → FC │ —                    │
  └─────────────┴──────────────────────────────────────────────────────────────────────────┴──────────────────────┘

  Event vocabulary (9 types): login_success, login_failed, password_change, add_payee, navigate_help, view_account, transfer, max_transfer, logout

  Fraud pattern: login_failed×N → password_change → add_payee → max_transfer
  Fraud rate in synthetic data: 12%

  ---
  Phase 7 — GNN Node Classification (src/gnn_trainer.py)

  Purpose: Graph-structural node embeddings for mule/fraud label classification

  Injected patterns for training:
  - Mule hubs (80 in-edges + 30 out-edges)
  - Synthetic identities (thin file + bust-out)
  - APP fraud (payee mismatch)
  - Smurfing rings (temporal cycles)

  Adversarial Robustness (src/adversarial_agent.py):
  - PPO agent generates evasion patterns (peel chains, smurfing, funnels, mule networks)
  - Continuous action space: peel %, wash intensity, cycle probability (Beta distributions)
  - State: 49-dim (32 graph embedding + 12 node features + 5 detection history)

  Phase 8 — Risk Consolidation (src/risk_consolidator.py)

  Weighted aggregation:

  ┌─────────────────┬────────┬───────────────────────────────────────────────────────┐
  │      Phase      │ Weight │                   Signals Included                    │
  ├─────────────────┼────────┼───────────────────────────────────────────────────────┤
  │ Spatial (Graph) │ 0.20   │ Cycles, fan-in/out, centrality                        │
  ├─────────────────┼────────┼───────────────────────────────────────────────────────┤
  │ Behavioral      │ 0.10   │ Cyber alerts                                          │
  ├─────────────────┼────────┼───────────────────────────────────────────────────────┤
  │ Temporal        │ 0.35   │ Volume acceleration, behavioral shifts, concentration │
  ├─────────────────┼────────┼───────────────────────────────────────────────────────┤
  │ LSTM            │ 0.25   │ Emerging link probabilities                           │
  ├─────────────────┼────────┼───────────────────────────────────────────────────────┤
  │ Cyber           │ 0.10   │ Secondary behavioral signals                          │
  └─────────────────┴────────┴───────────────────────────────────────────────────────┘

  Risk Tiers:
  - CRITICAL / HIGH: score > 0.7
  - MEDIUM: 0.4–0.7
  - LOW: < 0.4
  - CLEAN: 0.0

  Output per account: { spatial_score, behavioral_score, temporal_score, lstm_score, final_risk_score, risk_rank, signals[], signal_count }

  5. RISK WEIGHT OBSERVATIONS

  The current consolidation weights assign temporal signals 35% of total risk weight — the highest of any phase. This is intentional for catching acceleration patterns early, but it
  has a side effect: GBDT transaction-level scores and GNN embeddings are completely absent from the consolidator, meaning two of the eight pipeline phases contribute zero weight to
  the final risk score.

  Current effective weights (what actually reaches final score):
    Spatial  (Graph)    20%
    Behavioral          10%
    Temporal            35%
    LSTM                25%
    Cyber               10%
    GBDT                 0%  ← missing
    GNN                  0%  ← missing
    Sequence             0%  ← missing
